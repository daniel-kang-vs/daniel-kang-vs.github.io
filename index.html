Predicting NCAA Tournament Selection & Seeding

Problem: Can we predict which 68 teams make the NCAA tournament — and their seed — using only regular-season data?

  • Engineered 50 features from season stats, strength-of-schedule, and conference performance
  • Built a two-stage pipeline: Stage 1 classifies selection (yes/no), Stage 2 regresses seed for selected teams
  • Stacked 8 base models (HGB, GBM, ExtraTrees, RF) with a meta-learner; calibrated thresholds using PR curves

Results: Selection AUC = 0.966; seed MAE = 2.1 within-bracket accuracy.

Business Impact: Demonstrates end-to-end ML pipeline design: feature engineering, model stacking, threshold tuning, and interpretable output.

Shipping Box Standardization via Integer Programming

Problem: An e-commerce stroller retailer used a unique box per product, inflating dimensional-weight shipping costs and warehouse complexity.

  • Collected dimensions/weights for 17 top strollers and 45 candidate boxes
  • Formulated an ILP (Gurobi) minimizing total dimensional shipping weight while ensuring every stroller fits
  • Constrained the solver to select ≤7 box types for warehouse simplicity

Results: Reduced box catalog from 17 to 7 types with only 6.5% extra shipping cost vs. a perfect per-product match.

Business Impact: Warehouse picks faster, fewer SKUs to manage, and a clear cost-vs-complexity tradeoff for management.

Corporate Bankruptcy Early-Warning Model

Problem: Given 64 noisy financial ratios (40%+ missing), classify whether a firm will go bankrupt — a highly imbalanced binary task.

  • Winsorized extreme outliers and applied KNN imputation to preserve feature relationships
  • Trained XGBoost with stratified 5-fold CV; tuned scale_pos_weight for class imbalance

How I Approach Problems

Every analysis starts with a business question and ends with a recommendation

1

Frame the Question

Translate a stakeholder request into a testable hypothesis or measurable objective — the analysis is only useful if the question is right.

2

Get the Data Right

Pull, clean, and validate data. Handle missingness, outliers, and join logic before touching a model — garbage in, garbage out.

3

Model with Purpose

Choose the simplest method that answers the question: a well-designed A/B test, a regression, or a full ML pipeline when the problem demands it.

4

Validate Rigorously

Cross-validate, check assumptions, stress-test edge cases. A model that falls apart on new data is worse than no model.

5

Deliver the “So What”

Present findings as clear recommendations with quantified impact — not just charts and p-values, but what the business should do next.
